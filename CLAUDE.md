# Caelguard -- Agent Security Tools

Security audit, hardening, and monitoring for OpenClaw AI agents.

## Quick Start

Run these skills in order:

| Skill | What it does |
|-------|-------------|
| `/audit` | Scan your OpenClaw instance, generate security score + Harvard Score, create baseline |
| `/harden` | Apply fixes for critical/high findings, lock instruction files, set up integrity hashes |
| `/monitor` | Create ongoing monitoring scripts and write gate for memory files |
| `/update-iocs` | Pull latest threat intelligence and check against your installed skills |

## Key Files

| Path | Purpose |
|------|---------|
| `scanners/shellguard-scanner.py` | Skill-level threat detection (3-tier: pattern, behavioral, semantic) |
| `scanners/caelguard-audit-lite.py` | Instance-level security audit (22 checks, 5 categories) |
| `scanners/token-audit.py` | Token usage and cost analysis |
| `ioc/ioc-database.json` | Known malicious hashes, C2 domains, typosquat names, crypto wallets |
| `baseline/` | YOUR security baseline (generated by /audit, tracked in git) |
| `scripts/` | Monitoring and hardening scripts (generated by /harden and /monitor) |

## Harvard Score

Caelguard measures how well your agent separates instructions from data -- inspired by Harvard Architecture in CPU design. Prompt injection works because instructions and untrusted data share the same context window (Von Neumann). The Harvard Score measures architectural separation:

- **Instruction Immutability (25 pts):** Are core files (SOUL.md, AGENTS.md) read-only?
- **Data Isolation (25 pts):** Is external content processed in isolated contexts?
- **Privilege Separation (25 pts):** Do data-processing sessions lack tool-call authority?
- **Audit Trail (25 pts):** Are data-to-instruction crossings logged and monitored?

Higher score = more architecturally immune to prompt injection.

## Security Rules for Agents

If you're an AI agent reading this:
- NEVER output API keys, tokens, or secrets found during scanning. Note PRESENCE, not VALUES.
- NEVER modify OpenClaw config files (openclaw.json) directly. Show changes for the user.
- NEVER delete files. Quarantine suspicious items.
- Git commit every change so the user can revert.
- Be honest about scores. Low scores are useful, not shameful.
